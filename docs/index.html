Overview

Overview

Purpose

We wish to create a tool for Unity to allow hobbyist game developers to implement procedural generation in their games easily. Specifically, we wish to target developers creating Rogue-like indie games. The intended audience for this document are game developers, mostly those in the hobbyist scene, and also those who want to contribute to our open-source plugin. The purpose of this Software Requirements Specification (SRS) is to provide background knowledge pertaining to the context of our product, how it is to be used, and the expected output for given inputs.

The reader is assumed to have basic knowledge of game development and some foundational knowledge about procedural generation. Knowledge and understanding of UML diagrams are also required. The reader is also assumed to know about the Unity engine and its policies and agreements.

Scope

To give context, Unity is a cross-platform 2D/3D game engine with a powerful IDE for developers to create games. The toolkit software supports the development of content creation in Unity. The generation toolkit would procedurally generate 2D or 3D maps that game developers can use in their own games without having to implement their own procedural generation algorithms. This toolkit takes in inputs and parameters from the user, such as which algorithm to use, the randomness parameters, seed, assets/textures, etc.

Features

The Static Generation system feature consisted of two major use cases. Use case B: “Configuring and Parametrizing Procedural Generation Algorithms for Static Generation” (see Section 2.2.2) and use case C: “Generation for Static Content Creation” (see Section 2.2.3). Use case B involves the situation where the user picks PGA and fulfills its input of choice. This use case is to ensure that the data type and values of the chosen PGA are correct in order to successfully describe the characteristics of the generation.
