Scope

To give context, Unity is a cross-platform 2D/3D game engine with a powerful IDE for developers to create games. The toolkit software supports the development of content creation in Unity. The generation toolkit would procedurally generate 2D or 3D maps that game developers can use in their own games without having to implement their own procedural generation algorithms. This toolkit takes in inputs and parameters from the user, such as which algorithm to use, the randomness parameters, seed, assets/textures, etc., and generates randomized maps based on those inputs. To solve the decoupling problem, the toolkit would generate these maps (as a GameObject or FCAGO) in a way that is easily extendable to additional codes that would be built on top of it. Then, the Unity Editor Script communicates with native C# scripts that tell the system which implements the various procedural generation algorithms. This would be very useful for developers creating a new game because they can quickly generate maps with our toolkit and build the rest of the game logic using the maps we generate. This way, we provide a way to decouple the generation algorithm from the generated products, allowing the users the freedom to develop the actual interesting part of their game. Furthermore, the toolkit aims to develop a unique but playable generation.
